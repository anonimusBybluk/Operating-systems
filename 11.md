**1. Для чого використовують Git? Основні дії та команди**

**Git** — це система контролю версій. Її використовують для того, щоб:

- зберігати історію змін у проєкті;
- працювати над одним проєктом командою;
- повертатися до попередніх версій файлів;
- експериментувати з кодом, не ламаючи основну версію проєкту.

**Основні дії в Git**

1. **Створення або підключення репозиторію**\
   Репозиторій — це папка з проєктом, за якою “стежить” Git.
1. **Додавання файлів під контроль версій**\
   Git починає відслідковувати зміни у вибраних файлах.
1. **Збереження змін (комміти)**\
   Фіксація стану проєкту в певний момент часу.
1. **Робота з гілками (branches)**\
   Можна створювати окремі гілки для нових функцій або експериментів.
1. **Синхронізація з віддаленим репозиторієм (GitHub, GitLab)**\
   Надсилання змін у спільний репозиторій або отримання змін від інших.

**Основні команди Git**

- git init – створити новий репозиторій у папці
- git status – перевірити стан файлів (що змінено, що готове до комміту)
- git add . – додати всі зміни до індексації
- git commit -m "Опис змін" – зберегти зміни в історії
- git log – переглянути історію коммітів
- git branch – переглянути гілки
- git checkout -b new-branch – створити і перейти в нову гілку
- git pull – отримати оновлення з віддаленого репозиторію
- git push – відправити свої зміни на сервер

**2. Що таке комміт і як він допомагає відслідковувати зміни?**

**Комміт** — це зафіксований “знімок” стану файлів у певний момент часу.\
Кожен комміт має:

- унікальний ідентифікатор;
- автора;
- дату і час;
- опис змін (коментар);
- перелік змінених файлів.

**Як комміт допомагає відслідковувати зміни?**

Комміти дозволяють:

- бачити, **хто і що саме змінив**;
- переглядати історію змін по кожному файлу;
- порівнювати різні версії проєкту;
- повертатися до попереднього робочого стану, якщо щось зламалось.

**Короткий висновок**

Git допомагає контролювати процес розробки, не губити зміни та комфортно працювати в команді.\
Комміти — це основний механізм збереження історії проєкту та відстеження змін у файлах.
